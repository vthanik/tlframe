# ============================================================================
# arframe Decimal Alignment — Supported Statistical Display Formats
#
# The decimal alignment engine (`align = "decimal"`) recognizes 18 format
# families. Each column is auto-classified; the dominant type drives
# alignment. Mixed-type columns use family-aware fallbacks.
#
# This file is a user reference. See R/decimal.R for implementation.
# ============================================================================


# --------------------------------------------------------------------------
# 1. missing — Blank, dash, or token placeholders
# --------------------------------------------------------------------------
#
# Standalone missing values are replaced by spaces (same width as data).
#
# Examples:
#   (empty string)
#   -
#   --
#   ---
#   NA
#   NE   (Not Evaluable)
#   NC   (Not Calculable)
#   ND   (Not Done)
#   NR   (Not Reached)
#   BLQ  (Below Limit of Quantification)
#   INF
#   -INF


# --------------------------------------------------------------------------
# 2. n_only — Plain integer count
# --------------------------------------------------------------------------
#
# Right-aligned integers. Used in frequency columns.
#
# Examples:
#   0
#   42
#   135


# --------------------------------------------------------------------------
# 3. scalar_float — Signed decimal number
# --------------------------------------------------------------------------
#
# Aligned on decimal point. Integer part right-aligned, decimals left-aligned.
#
# Examples:
#   12.3
#   135.20
#   -2.5
#   0.07


# --------------------------------------------------------------------------
# 4. pvalue — Prefixed p-value
# --------------------------------------------------------------------------
#
# Supports <, >, = prefixes. Decimal dot aligned.
#
# Examples:
#   <0.001
#   =0.500
#   >0.999


# --------------------------------------------------------------------------
# 5. n_pct — Count (percentage)
# --------------------------------------------------------------------------
#
# n right-aligned, percentage decimal-aligned inside parens.
# n=0 suppresses the percentage part (Style D zero suppression).
#
# Examples:
#   0
#   1 (2.2)
#   42 (50.0%)
#   100 (100.0%)


# --------------------------------------------------------------------------
# 6. n_over_N_pct — Fraction with percentage: n/N (pct)
# --------------------------------------------------------------------------
#
# Numerator right-aligned, denominator left-aligned, pct decimal-aligned.
#
# Examples:
#   3/45 (6.7)
#   42/84 (50.0%)
#   120/120 (100.0)


# --------------------------------------------------------------------------
# 7. n_over_N — Plain fraction (no percentage)
# --------------------------------------------------------------------------
#
# Style D zeros, response counts without percentage.
#
# Examples:
#   0/120
#   1/120
#   108/120


# --------------------------------------------------------------------------
# 8. n_over_float — Events over subject-time
# --------------------------------------------------------------------------
#
# Integer numerator / decimal denominator. Exposure-adjusted rates.
#
# Examples:
#   0/234.6
#   12/234.6
#   108/234.6


# --------------------------------------------------------------------------
# 9. est_spread — Estimate (spread)
# --------------------------------------------------------------------------
#
# Mean (SD), Median (IQR), etc. Both parts decimal-aligned.
#
# Examples:
#   75.0 (6.75)
#   136.8 (17.61)
#   -0.0 (1.47)


# --------------------------------------------------------------------------
# 10. est_spread_pct — Estimate (CV%)
# --------------------------------------------------------------------------
#
# Geometric mean with coefficient of variation. Spread has % suffix.
#
# Examples:
#   0.10 (8.7%)
#   52.43 (23.4%)
#   1240.40 (23.4%)


# --------------------------------------------------------------------------
# 11. est_ci — Estimate (lo, hi) with token support
# --------------------------------------------------------------------------
#
# Point estimate with confidence interval in parentheses.
# Each position (est, lo, hi) can be a number OR a missing token
# (NR, NE, INF, BLQ, etc.) for censored/incomplete data.
#
# Examples:
#   168.0 (152.4, 183.6)
#   1.520 (0.650, 3.570)
#   -2.1 (-4.3, 0.1)
#   14.3 (11.2, NR)
#   NR (NR, NR)
#   0.087 (0.034, NR)
#   3.120 (1.840, INF)


# --------------------------------------------------------------------------
# 12. est_ci_bracket — Estimate [lo, hi] with token support
# --------------------------------------------------------------------------
#
# Same as est_ci but with square bracket delimiters.
#
# Examples:
#   0.0 [0.0, 0.0]
#   53.0 [45.0, 60.0]
#   102.0 [88.4, 116.2]


# --------------------------------------------------------------------------
# 13. range_pair — Comma-separated range: lo, hi
# --------------------------------------------------------------------------
#
# Min, Max or similar. Both sides decimal-aligned.
#
# Examples:
#   2.0, 45.0
#   65.0, 88.0
#   -5.3, 12.1


# --------------------------------------------------------------------------
# 14. int_range — Integer range with dash: lo - hi
# --------------------------------------------------------------------------
#
# Day ranges, visit windows, etc.
#
# Examples:
#   1 - 180
#   10 - 365


# --------------------------------------------------------------------------
# 15. est_ci_pval — CI + trailing p-value (compound)
# --------------------------------------------------------------------------
#
# Full efficacy line: estimate (CI) followed by p-value.
# CI positions and p-value can be missing tokens.
#
# Examples:
#   -0.08 (-0.21, 0.05)    0.194
#   12.40 (9.80, 15.00)    <0.001
#   3.70 (1.24, 6.16)    0.003


# --------------------------------------------------------------------------
# 16. n_pct_rate — Count (%) + trailing rate (compound)
# --------------------------------------------------------------------------
#
# SAE tables: n (%) with incidence rate per 100 PY.
#
# Examples:
#   0 (0.0)    0.00
#   3 (2.5)    1.28
#   42 (35.0)    17.94


# --------------------------------------------------------------------------
# 17. n_over_N_pct_ci — n/N (%) [CI] (compound)
# --------------------------------------------------------------------------
#
# Response rate tables: fraction with percentage and bracket CI.
#
# Examples:
#   0/120 (0.0) [0.0, 3.0]
#   12/120 (10.0) [5.6, 16.9]
#   120/120 (100.0) [97.0, 100.0]


# --------------------------------------------------------------------------
# 18. est_spread_pct_ci — Gmean (CV%) (CI) (compound)
# --------------------------------------------------------------------------
#
# PK summary lines: geometric mean with CV% and confidence interval.
#
# Examples:
#   8.1 (24.2%)    (7.3, 8.9)
#   1240.4 (23.4%)    (1124.2, 1368.8)


# ============================================================================
# Cross-type fallback behavior
# ============================================================================
#
# When a column mixes types, the dominant type sets the alignment grid.
# Minority types use these fallback rules:
#
#   n_only          -> aligns to integer part of any numeric type
#   scalar_float    -> aligns decimal with estimate in est_* types
#   pvalue          -> aligns decimal with est or scalar_float
#   n_pct           -> aligns n and parens with n_over_N_pct
#   est_spread      -> aligns est part with est_ci; spread with lo bound
#   est_ci          <-> est_ci_bracket (swap delimiters, same widths)
#   est_spread_pct  <-> est_spread (handle % presence/absence)
#   n_over_N        -> aligns fraction with n_over_N_pct (no pct)
#   range_pair      -> aligns left value with est part in est_* types
#   missing/unknown -> replaced with spaces (full width)
#
# All other cross-type combinations use generic right-padding.
